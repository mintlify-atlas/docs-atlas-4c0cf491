## Overview

Neru has a comprehensive test suite with clear separation between unit tests and integration tests. All new code requires appropriate test coverage.

<Note>
  Tests must pass before code can be merged. Run `just test` to verify all tests pass.
</Note>

## Test Organization

### Test Types

| Test Type | File Pattern | Purpose | Command | Build Tag |
|-----------|--------------|---------|---------|------------|
| **Unit Tests** | `*_test.go` | Business logic with mocks, no system dependencies | `just test-unit` | None |
| **Integration Tests** | `*_integration_darwin_test.go`<br/>`*_integration_linux_test.go` | Real system interactions with platform APIs | `just test-integration` | `integration` |

### File Naming Convention

```text
package_test.go                          # Unit tests (logic, mocks)
package_integration_darwin_test.go       # macOS integration tests
package_integration_linux_test.go        # Linux integration tests
```

## Unit Tests

### Purpose

Unit tests verify business logic in isolation using mocks for external dependencies. They should be:

- **Fast**: No I/O, no system calls
- **Isolated**: Test one unit of functionality
- **Deterministic**: Same input always produces same output
- **Independent**: Can run in any order

### Table-Driven Tests

Use table-driven tests for testing multiple scenarios:

```go
func TestHintGenerator_Generate(t *testing.T) {
    tests := []struct {
        name        string
        charset     string
        elementCount int
        want        int
        wantErr     bool
    }{
        {
            name:        "generates hints for small set",
            charset:     "asdf",
            elementCount: 10,
            want:        10,
            wantErr:     false,
        },
        {
            name:        "generates hints for large set",
            charset:     "abcdefghij",
            elementCount: 100,
            want:        100,
            wantErr:     false,
        },
        {
            name:        "fails with empty charset",
            charset:     "",
            elementCount: 10,
            want:        0,
            wantErr:     true,
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            gen := NewGenerator(tt.charset, testLogger())
            hints, err := gen.Generate(tt.elementCount)

            if (err != nil) != tt.wantErr {
                t.Errorf("Generate() error = %v, wantErr %v", err, tt.wantErr)
                return
            }

            if len(hints) != tt.want {
                t.Errorf("Generate() got %d hints, want %d", len(hints), tt.want)
            }
        })
    }
}
```

### Using Mocks

Use mocks to isolate the unit under test:

```go
type mockAccessibilityPort struct {
    elements []*domain.Element
    err      error
}

func (m *mockAccessibilityPort) GetElements() ([]*domain.Element, error) {
    return m.elements, m.err
}

func TestHintService_Generate(t *testing.T) {
    // Setup mock
    mockElements := []*domain.Element{
        {Role: "button", Bounds: domain.Rect{X: 0, Y: 0, Width: 100, Height: 50}},
        {Role: "link", Bounds: domain.Rect{X: 100, Y: 0, Width: 80, Height: 30}},
    }
    mockAcc := &mockAccessibilityPort{elements: mockElements}

    // Create service with mock
    service := NewHintService(mockAcc, nil, nil, nil, nil, testLogger())

    // Test
    hints, err := service.GenerateHints()
    if err != nil {
        t.Fatalf("GenerateHints() error = %v", err)
    }

    if len(hints) != len(mockElements) {
        t.Errorf("got %d hints, want %d", len(hints), len(mockElements))
    }
}
```

## Integration Tests

### Purpose

Integration tests verify that components work correctly with real system APIs. They:

- **Use Real APIs**: Test actual platform functionality
- **Are Platform-Specific**: Tagged with OS build constraints
- **May Be Slower**: Involve I/O and system calls
- **Require System Resources**: Need accessibility permissions, windows, etc.

### Platform-Specific Tests

Use build tags to specify platform requirements:

```go
//go:build integration && darwin

package accessibility

import (
    "testing"
    "github.com/y3owk1n/neru/internal/core/infra/accessibility"
)

func TestAccessibilityAdapter_GetElementAtPosition_Integration(t *testing.T) {
    // Skip if not running in CI or without proper permissions
    if testing.Short() {
        t.Skip("Skipping integration test in short mode")
    }

    adapter := accessibility.NewAdapter(testLogger())
    element, err := adapter.GetElementAtPosition(100, 100)

    if err != nil {
        t.Fatalf("GetElementAtPosition() error = %v", err)
    }

    if element == nil {
        t.Error("expected element, got nil")
    }
}
```

        controller := ipc.NewController(socketPath, handler, logger)

    func TestEventTap_Integration(t *testing.T) {
        // Set timeout for slow operations
        ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
        defer cancel()
        
        eventTap := eventtap.New(logger)
        err := eventTap.Start(ctx)
        if err != nil {
            t.Fatalf("Start() error = %v", err)

        defer eventTap.Stop()

        case <-ctx.Done():

        }
    }

## Test Best Practices

### Test Structure

Follow the Arrange-Act-Assert pattern:

```go
func TestExample(t *testing.T) {
    // Arrange: Set up test conditions
    input := "test input"
    expected := "expected output"
    service := NewService(testDependencies())
    
    // Act: Execute the code under test
    result, err := service.Process(input)
    
    // Assert: Verify the results
    if err != nil {
        t.Fatalf("Process() error = %v", err)
    }
    if result != expected {
        t.Errorf("Process() = %v, want %v", result, expected)
    }
}
```

### Error Testing

<Tabs>
  <Tab title="Error Conditions">
    ```go
    func TestService_Error(t *testing.T) {
        service := NewService(nil)  // Invalid config
        
        _, err := service.Process("input")
        if err == nil {
            t.Error("expected error, got nil")
        }
        
        // Test specific error type
        if !errors.Is(err, ErrInvalidConfig) {
            t.Errorf("expected ErrInvalidConfig, got %v", err)
        }
    }
    ```
  </Tab>
  
  <Tab title="Error Messages">
    ```go
    func TestValidator_Validate(t *testing.T) {
        validator := NewValidator()
        
        err := validator.Validate(invalidInput)
        if err == nil {
            t.Fatal("expected error, got nil")
        }
        
        // Check error message contains useful information
        errMsg := err.Error()
        if !strings.Contains(errMsg, "field") {
            t.Errorf("error message should mention field name, got: %s", errMsg)
        }
    }
    ```
  </Tab>
</Tabs>

### Test Helpers

Create helper functions for common test setup:

```go
// testLogger returns a no-op logger for tests
func testLogger() *zap.Logger {
    return zap.NewNop()
}

// testConfig returns a valid test configuration
func testConfig() *config.Config {
    return &config.Config{
        Hints: config.HintsConfig{
            Charset: "asdfghjkl",
        },
        // ... other defaults
    }
}

// testElement creates a test element with given properties
func testElement(role string, x, y, width, height int) *domain.Element {
    return &domain.Element{
        Role: role,
        Bounds: domain.Rect{
            X:      x,
            Y:      y,
            Width:  width,
            Height: height,
        },
    }
}
```

## Debugging Tests

### Verbose Output

```bash
# Run tests with verbose output
go test -v ./...

# Run specific test with verbose output
go test -v -run TestHintGenerator_Generate ./internal/core/domain/hint/
```

### Test-Specific Logging

```go
func TestDebugExample(t *testing.T) {
    // Enable debug logging for this test
    logger := zaptest.NewLogger(t)
    service := NewService(logger)
    
    // Logs will appear in test output
    result, err := service.Process("input")
    if err != nil {
        t.Fatalf("Process() error = %v", err)
    }
}
```

### Debugging Individual Tests

```bash
# Run single test
go test -run TestSpecificName ./pkg/

# Run with debugging
dlv test ./internal/app/services/ -- -test.run TestHintService
```

## Common Test Patterns

<AccordionGroup>
  <Accordion title="Testing with contexts">
    ```go
    func TestWithContext(t *testing.T) {
        ctx, cancel := context.WithTimeout(context.Background(), 1*time.Second)
        defer cancel()
        
        result, err := DoSomethingWithContext(ctx)
        if err != nil {
            t.Fatalf("unexpected error: %v", err)
        }
    }
    ```
  </Accordion>
  
  <Accordion title="Testing concurrent code">
    ```go
    func TestConcurrent(t *testing.T) {
        var wg sync.WaitGroup
        errors := make(chan error, 10)
        
        for i := 0; i < 10; i++ {
            wg.Add(1)
            go func() {
                defer wg.Done()
                if err := DoWork(); err != nil {
                    errors <- err
                }
            }()
        }
        
        wg.Wait()
        close(errors)
        
        for err := range errors {
            t.Errorf("concurrent error: %v", err)
        }
    }
    ```
  </Accordion>
  
  <Accordion title="Testing with temporary files">
    ```go
    func TestFileOperations(t *testing.T) {
        // Create temp directory (automatically cleaned up)
        tmpDir := t.TempDir()
        testFile := filepath.Join(tmpDir, "test.txt")
        
        // Write test data
        err := os.WriteFile(testFile, []byte("test"), 0644)
        if err != nil {
            t.Fatalf("failed to write test file: %v", err)
        }
        
        // Test your code that reads the file
        result, err := ReadConfig(testFile)
        if err != nil {
            t.Errorf("ReadConfig() error = %v", err)
        }
    }
    ```
  </Accordion>
</AccordionGroup>
