## Prerequisites

<Steps>
  <Step title="Install Neru">
    Install via Homebrew (recommended):

    ```bash
    brew tap y3owk1n/tap
    brew install --cask y3owk1n/tap/neru
    ```

    See [Installation](/installation) for other methods.
  </Step>

  <Step title="Grant accessibility permissions">
    1. Open **System Settings → Privacy & Security → Accessibility**
    2. Click **+** and add **Neru**
    3. Enable the toggle next to Neru
  </Step>

  <Step title="Launch Neru">
    ```bash
    open -a Neru
    ```

    Or install it to start automatically:

    ```bash
    neru services install
    ```
  </Step>
</Steps>

## Try Your First Mode

Neru provides three navigation modes. Let's start with **Recursive Grid Mode** — the recommended mode that works everywhere.

### What is Recursive Grid Mode?

**Recursive Grid Mode** divides your screen into a coordinate grid. You drill into any region with `u`/`i`/`j`/`k` until you land exactly where you want.

- Works in **every app, every window, every corner of macOS**
- No exceptions, no app-specific setup
- Precise and predictable

### Activate Recursive Grid Mode

Press the default hotkey:

<Card title="Cmd+Shift+C" icon="grid-2">
  Activates Recursive Grid Mode
</Card>

You'll see a coordinate grid overlay on your screen.

### Navigate with the Grid

The grid divides your screen into labeled regions. Navigate using these keys:

| Key | Action |
| --- | ------ |
| `u` | Select **upper-left** quadrant |
| `i` | Select **upper-right** quadrant |
| `j` | Select **lower-left** quadrant |
| `k` | Select **lower-right** quadrant |

<Steps>
  <Step title="Type the label">
    The grid shows labels for each region. Type the label (e.g., `u`, `i`, `j`, `k`) to drill into that region.
  </Step>

  <Step title="Repeat to narrow down">
    The grid subdivides recursively. Keep typing labels until you reach your target.
  </Step>

  <Step title="Click">
    Once you're over your target, press `Enter` to click.
  </Step>
</Steps>

<Note>
  To cancel the mode at any time, press `Esc`.
</Note>

## Try Other Modes

Neru provides three navigation modes:

<CardGroup cols={3}>
  <Card title="Recursive Grid" icon="grid-2">
    **Cmd+Shift+C** (recommended)
    
    Precise coordinate-based navigation that works everywhere.
  </Card>
  <Card title="Grid" icon="grid">
    **Cmd+Shift+G**
    
    Same approach without recursive subdivision. Great for quick coarse navigation.
  </Card>
  <Card title="Hints" icon="tags">
    **Cmd+Shift+Space**
    
    Labels appear on every clickable element. Requires accessibility tree support.
  </Card>
</CardGroup>

### Hints Mode

**Hints Mode** labels every clickable element on screen.

<Steps>
  <Step title="Activate Hints Mode">
    Press **Cmd+Shift+Space**
  </Step>

  <Step title="Type the label">
    Labels appear on clickable elements. Type the label to click that element.
  </Step>
</Steps>

<Warning>
  Hints Mode requires accessibility tree support. Some apps may need additional configuration. See [Troubleshooting](/advanced/troubleshooting) for app-specific notes.
</Warning>

### Grid Mode

**Grid Mode** provides a static coordinate grid without recursive subdivision.

<Steps>
  <Step title="Activate Grid Mode">
    Press **Cmd+Shift+G**
  </Step>

  <Step title="Type coordinates">
    Type the two-character label to move the cursor to that position.
  </Step>

  <Step title="Click">
    Press `Enter` to click at the cursor position.
  </Step>
</Steps>

## Direct Mouse Actions

In **any navigation mode**, you can trigger mouse actions without switching modes:

| Hotkey | Action |
| ------ | ------ |
| `Shift+L` | **Left click** |
| `Shift+R` | **Right click** |
| `Shift+D` | **Double click** |
| `Shift+M` | **Middle click** |

These actions work in Recursive Grid, Grid, and Hints modes.

## Scroll Mode

Neru provides Vim-style scrolling with familiar keybindings.

<Steps>
  <Step title="Activate Scroll Mode">
    Press **Cmd+Shift+S**
  </Step>

  <Step title="Scroll with Vim keys">
    | Key | Action |
    | --- | ------ |
    | `j` | Scroll down |
    | `k` | Scroll up |
    | `gg` | Scroll to top |
    | `G` | Scroll to bottom |
    | `Ctrl+D` | Scroll down half page |
    | `Ctrl+U` | Scroll up half page |
  </Step>

  <Step title="Exit Scroll Mode">
    Press `Esc` to exit.
  </Step>
</Steps>

<Note>
  All scroll keybindings are fully remappable in the configuration file.
</Note>

## Per-App Exclusions

You can disable Neru in specific apps where you don't need it.

Edit `~/.config/neru/config.toml`:

```toml config.toml
[general]
excluded_apps = [
  "com.apple.Terminal",
  "com.googlecode.iterm2",
]
```

Neru will be disabled in Terminal and iTerm2.

### Why Recursive Grid?

> "I started with hints mode (like Vimium or Homerow), where accessibility drives everything. But accessibility doesn't cover all cases, especially for Electron apps. There's a need for workarounds for Firefox, Chromium, and Electron-based apps. Even macOS native apps sometimes lack accessibility access.
>
> Then I wondered if there's a better way to navigate with the keyboard without complex accessibility edge cases. The answer is grid (like Warpd or Mouseless), which provides a static grid. Now we don't have to care about accessibility access or targeted delays. I'm now fully in grid mode even when I still support accessibility."
>
> — Kyle Wong (author of Neru)
